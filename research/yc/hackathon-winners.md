## Common Patterns in Winning Projects

### Complexity

- Scoped, focused MVPs (1-3 core MCP integrations/tools)
- Agentic (LLM orchestrates data/RAG/actions) but not over-scoped
- Heavy on real external data/APIs vs pure mocks
- Technical depth in secure/reliable MCP servers
- Always demoable in <24-48h

### Polish

- High — clean web dashboards (Next.js/Vercel), conversational UIs
- Visualizations (weather metaphors, animated flows, health scores, astrology reports)
- Chrome extensions, Gradio apps
- Deployed live, professional aesthetics
- "Delightful UX" is an explicit prize category in multiple events

### Narrative

- Strong "pain → solution → impact" story
- Real-world problems solved autonomously/actionably
- Often with metrics (adoption %, time saved, health scores) or "aha" moments
- Make it relatable to judges' worlds

**Winners ship functional agents/tools with real MCP data flows wrapped in polished, story-driven interfaces. Rough but deep tech rarely wins over scoped + impressive demo.**

## What Judges Typically Value

| Factor                       | Weight      | Notes                                                          |
| ---------------------------- | ----------- | -------------------------------------------------------------- |
| **Novelty/Creative MCP use** | High        | New domains (voice, viz, games, critiques) + usefulness        |
| **Technical execution**      | High        | Working, robust integrations. Live data > mock for credibility |
| **Business viability**       | High        | Scalable ideas, revenue signals, "YC-fit"                      |
| **Demo/UX/polish**           | **Highest** | Clear, smooth, engaging. "Delightful" trumps raw complexity    |

General consensus: **Working prototype > half-baked ambitious idea. Polish + narrative often edges out pure tech depth. Live/real data shines for authenticity.**

## Demo Techniques That Land

1. **Live data wins big** — Real-time integrations build credibility and "wow." Hybrid (live core + graceful mock fallback) is common in top entries.
2. **Tight narrative arc** — "Here's the pain" → quick core demo → impact with metrics. Make it relatable.
3. **1-3 min focused** — Nail 1-2 killer features. Rehearse for error-free flow. Video backup if live is risky.
4. **Interactive where possible** — Let judges test the dashboard/agent.
5. **Deployed URL** — End-to-end flow; "aha" moments; quantifiable outcomes.
